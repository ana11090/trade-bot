# MT5 EA Generator Fix - Metals Pip Scaling

**Date:** 2026-04-26
**Issue:** EA generator produces 10x smaller SL/TP values for metals (XAUUSD, XAGUSD)
**Status:** ✅ FIXED

## Problem Summary

When generating MT5 EAs for metal instruments (XAUUSD, XAGUSD), the EA generator was using pip values designed for forex pairs, resulting in:

- **SL too tight:** 150 pips = 1.5 points (should be 1500 pips = 15 points)
- **TP too tight:** 300 pips = 3 points (should be 3000 pips = 30 points)
- **Result:** Every trade immediately hit stop loss in MT5

### Root Cause

**Forex vs Metals Pip Calculation:**

| Instrument | Digits | Point | Pip Size | 150 pips = |
|------------|--------|-------|----------|------------|
| EURUSD | 5 | 0.00001 | 0.0001 | 0.015 points |
| XAUUSD | 2 | 0.01 | 0.01 | 1.5 points ❌ |
| XAUUSD (correct) | 2 | 0.01 | 0.01 | **15 points** ✓ |

For XAUUSD:
- 1 pip = 0.01 (one cent)
- ATR on H1 = ~10-15 points = 1000-1500 pips
- Backtest SL = 1.5 × ATR = 15 points = **1500 pips**
- But EA generator was outputting = **150 pips** (10x too small!)

### Scaling Logic:

```python
# Detect instrument type
from shared.prop_firm_engine import get_instrument_type
_inst_type = get_instrument_type(symbol)  # Returns 'metals', 'forex', or 'indices'

# Scale pip values for metals
if _inst_type == 'metals':
    if sl_pips > 0 and sl_pips < 1000:
        sl_pips *= 10  # 150 → 1500
    if tp_pips > 0 and tp_pips < 1000:
        tp_pips *= 10  # 300 → 3000
    if trail_activation_pips > 0 and trail_activation_pips < 1000:
        trail_activation_pips *= 10  # 50 → 500
    if trail_distance_pips > 0 and trail_distance_pips < 1000:
        trail_distance_pips *= 10  # 100 → 1000
    if breakeven_pips > 0 and breakeven_pips < 1000:
        breakeven_pips *= 10  # 50 → 500
```

### Safety Check:

Only scales values **< 1000** to avoid double-scaling if user manually entered correct values.
